Generated by /plan-eng-review on 2026-03-21

## Affected Pages/Routes
- /onboarding — 5-step flow: welcome, role, skills, experience, sources
- /radar — Dashboard with today's brief + skill gap list
- /feed — Tabbed article feed (All/Reddit/HN/Blogs) with SKILL GAP badges
- /profile — Edit role, skills, feed sources

## Key Interactions to Verify
- Onboarding chip selection (single + multi-select) saves to Firestore
- Feed tab filtering shows correct articles per source
- SKILL GAP badge appears on articles tagged by Claude analysis
- Pull-to-refresh triggers Firestore re-read
- Feed pagination loads 20 articles, then more on scroll
- Dashboard shows empty state when 0 gaps detected

## Edge Cases
- Offline mode: app opens with cached data, shows "No connection" on refresh
- 0 articles fetched (all sources down): dashboard shows cached brief with stale date
- Claude returns invalid JSON: retry once, then show cached brief
- RSS 429 rate limit: skip source, show partial results
- Empty onboarding (no skills selected): should still work with empty skills array

## Critical Paths (must-have tests)
1. Trend label computation: New/Rising/Hot classification from historical SkillGap data
2. RSS article deduplication: same URL should not create duplicate Firestore documents
3. Claude JSON parsing + retry: valid JSON → success, invalid JSON → retry, API error → cached brief
4. Firestore security rules: user can only read/write their own data, articles are read-only for clients
