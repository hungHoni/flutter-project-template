## Architecture Notes

### File Structure
```
lib/
├── main.dart                          # App entry, ProviderScope, MaterialApp.router
├── app/
│   ├── router.dart                    # GoRouter config with all routes
│   └── app.dart                       # MaterialApp wrapper with theme
├── theme/
│   ├── app_colors.dart                # 11 color tokens as static const Color
│   ├── app_typography.dart            # 8 text style tokens
│   ├── app_spacing.dart               # 7 spacing tokens as static const double
│   ├── app_theme.dart                 # ThemeData factory (overrides all Material defaults)
│   └── app_icons.dart                 # Phosphor icon mappings
├── features/
│   ├── onboarding/
│   │   └── screens/
│   │       └── onboarding_screen.dart # Placeholder with PageView for 5 steps
│   ├── radar/
│   │   └── screens/
│   │       └── radar_screen.dart      # Dashboard placeholder
│   ├── feed/
│   │   └── screens/
│   │       └── feed_screen.dart       # Feed placeholder
│   └── profile/
│       └── screens/
│           └── profile_screen.dart    # Profile placeholder
└── shared/
    └── widgets/
        └── app_bottom_nav.dart        # Custom bottom navigation bar
```

### Key Decisions
- **GoRouter** for declarative routing (standard for Flutter, supports deep links)
- **Feature-first folder structure** (not layer-first) — scales better as screens grow
- **Custom BottomNavigationBar** — the design system prohibits default Material `BottomNavigationBar`
- **No `AppBar`** — screens use custom headers per the minimalist design system
- **Fonts:** Georgia (serif) for display, system sans-serif for body, Courier New (mono) for labels. Use `google_fonts` for DM Sans and Instrument Serif as recommended by the design system skill, falling back to Georgia and system fonts.
